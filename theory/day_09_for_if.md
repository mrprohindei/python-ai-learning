# День 9 — цикл `for` и `if` вместе

## Зачем это придумали

**Проблема:** в реальных задачах редко «просто повтори 10 раз одно и то же». Чаще: «пройди по **каждому** элементу, но **для одних** сделай A, **для других** B» — пропусти, посчитай, отфильтруй.

**Зачем совмещать:** цикл даёт **массовость** (обработать все числа, все строки), `if` — **выбор** на каждом шаге. Так работают фильтры, подсчёт статистики, поиск ошибок в данных.

---

## Зачем совмещать

В день 8 цикл **всегда** делал одно и то же на каждом шаге — например, печатал каждое число.

Часто нужно иначе: **на одних шагах — одно действие, на других — другое** или **пропустить** шаг.

Пример: вывести только числа больше 5. Или посчитать, сколько чисел чётные.

Для этого внутрь цикла ставят **`if`** — то, что ты уже знаешь с дня 4.

---

## `if` внутри `for`

```python
for i in range(1, 11):
    if i > 5:
        print(i)
```

Вывод: `6`, `7`, `8`, `9`, `10` — единицы и пятёрка не печатаются.

Как читать:

1. Цикл перебирает `i` от 1 до 10.
2. **На каждом шаге** проверяется условие `i > 5`.
3. `print` срабатывает **только** когда условие истинно.

---

## Отступы: два уровня

```python
for i in range(3):      # уровень 1 — цикл
    if i > 0:           # уровень 2 — внутри цикла
        print(i)        # уровень 3 — внутри if
```

| Строка | Где выполняется |
|--------|-----------------|
| `for` | начало цикла |
| `if` с отступом | на **каждом** шаге цикла |
| `print` с двойным отступом | только когда `if` сработал |

`print` **без** отступа под `for` — **вне** цикла, один раз в конце.

Частая ошибка: `if` и `print` на одном уровне — тогда `print` не внутри `if`.

---

## `if` / `else` внутри цикла

```python
for i in range(1, 4):
    if i % 2 == 0:
        print(i, "— чётное")
    else:
        print(i, "— нечётное")
```

На каждом шаге — **либо** ветка `if`, **либо** `else`.

---

## Оператор `%` — остаток от деления

Новое на этом уроке:

```python
7 % 2   # → 1  (7 = 3×2 + 1)
8 % 2   # → 0  (8 = 4×2 + 0)
10 % 3  # → 1
```

**Чётное число** — делится на 2 без остатка:

```python
if i % 2 == 0:
    print(i, "чётное")
```

| Выражение | Когда истинно |
|-----------|----------------|
| `i % 2 == 0` | `i` чётное |
| `i % 2 != 0` | `i` нечётное |

Пока достаточно `% 2` для чётности. Другие делители — позже.

## Накопление + условие

В день 8 ты считал **сумму** всех чисел. Можно считать только **часть** из них.

**Счётчик** — сколько раз условие выполнилось:

```python
count = 0
for i in range(1, 11):
    if i > 5:
        count = count + 1
print("Больше пяти:", count)
```

Та же схема, что `z = z + i`, но прибавляем `1`, когда условие подходит.

**Произведение** (умножение всех чисел от 1 до N):

```python
n = 5
product = 1          # для умножения старт = 1, не 0!
for i in range(1, n + 1):
    product = product * i
print(product)       # 120
```

| Что считаем | Старт до цикла | В цикле |
|-------------|----------------|---------|
| сумма | `0` | `total = total + i` |
| произведение | `1` | `product = product * i` |
| счётчик | `0` | `count = count + 1` |

**Почему для произведения `1`, а не `0`?**  
`0 * что угодно` всегда `0` — результат испортится.

## Что уже можно комбинировать

Внутри одного цикла:

- `input` — обычно **до** цикла (спросить N один раз)
- `for` + `range`
- `if` / `else` + сравнения
- `int()`, арифметика, `print`
- накопление в переменной

---

## Частые ошибки

| Ошибка | Что происходит |
|--------|----------------|
| `if` без отступа под `for` | `SyntaxError` или логика ломается |
| `print` на уровне `for`, а не `if` | печатает всё, условие игнорируется |
| `product = 0` перед умножением | результат всегда 0 |
| забыли `range(1, n + 1)` | последнее число не входит |

---

## Мини-шпаргалка

```python
# только числа > 5
for i in range(1, 11):
    if i > 5:
        print(i)

# чётные от 2 до 10
for i in range(2, 11):
    if i % 2 == 0:
        print(i)

# произведение 1..n
n = 4
p = 1
for i in range(1, n + 1):
    p = p * i
```
