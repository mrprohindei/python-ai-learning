# День 8 — цикл `for` и `range`

## Зачем это придумали

**Проблема:** компьютер силён в **повторении**. Писать одно и то же руками (`print(1)` … `print(100)`) — долго, легко ошибиться, невозможно, если число шагов заранее неизвестно (1000 пользователей, 50 файлов).

**Зачем цикл:** сказать «сделай это N раз» или «пройди по каждому элементу» — одной конструкцией. Так работают обработка списков, пикселей, сообщений в чате.

**`range`** — способ сказать «N шагов с числами», не создавая длинный список вручную.

---

## Зачем нужны циклы

Без цикла, чтобы вывести числа 1–10, пришлось бы писать десять раз `print`:

```python
print(1)
print(2)
# ... ещё 8 строк
```

Цикл говорит Python: **повтори этот блок несколько раз** — с разными значениями.

---

## Синтаксис `for`

```python
for переменная in последовательность:
    тело цикла
```

- **`for`** — начало цикла
- **`переменная`** — имя, которое на каждом шаге получает новое значение
- **`in`** — «перебирай по очереди»
- **`последовательность`** — что перебираем (часто `range(...)`)
- **`:`** — как у `if`, дальше идёт блок
- **отступ** — всё с отступом выполняется **на каждом** шаге цикла

Пример:

```python
for i in range(3):
    print(i)
```

Вывод:

```
0
1
2
```

На первом шаге `i = 0`, на втором `i = 1`, на третьем `i = 2`. Потом цикл **заканчивается**.

---

## Функция `range`

`range` создаёт **набор целых чисел** для перебора.

| Вызов | Что перебирается |
|-------|------------------|
| `range(5)` | 0, 1, 2, 3, 4 |
| `range(1, 6)` | 1, 2, 3, 4, 5 |
| `range(0, 10, 2)` | 0, 2, 4, 6, 8 |

Правило: `range(начало, конец)` — **конец не входит**.  
Поэтому «до 10 включительно» → `range(1, 11)`.

---

## Как читать код с циклом

Разберём:

```python
for n in range(1, 4):
    print("Число:", n)
```

Пошагово:

1. Python создаёт числа 1, 2, 3
2. `n = 1` → печатает `Число: 1`
3. `n = 2` → печатает `Число: 2`
4. `n = 3` → печатает `Число: 3`
5. Числа кончились — цикл завершён, программа идёт дальше

---

## Цикл и отступы

```python
for i in range(3):
    print(i)
print("Готово")
```

`print("Готово")` **без** отступа — **вне** цикла. Выполнится **один раз**, после всех повторов.

Частая ошибка: забыть отступ — код не попадёт в цикл или будет `IndentationError`.

---

## `for` + то, что уже знаешь

Внутри цикла можно всё из прошлых дней:

```python
for i in range(1, 4):
    x = i * 10
    print(x)
```

`input`, `if`, арифметика — работают **внутри** тела цикла так же, как снаружи.

---

## Частые ошибки

| Ошибка | Что происходит |
|--------|----------------|
| `range(1, 10)` вместо `range(1, 11)` | нет десятки — конец не входит |
| нет отступа у `print` внутри `for` | ошибка или код вне цикла |
| путают `i` и значение | `i` — просто имя, можно `for x in range(5)` |

---

## Мини-шпаргалка

```python
# числа 1..10
for i in range(1, 11):
    print(i)

# n раз повторить текст
for _ in range(5):
    print("Привет")
```

`_` — когда номер шага не нужен, важен только **счёт повторов**.
