# День 11 — цикл `while`

## Зачем это придумали

**Проблема:** не всегда знаешь, **сколько раз** повторить. Пользователь может ошибиться 1 раз или 10; игра идёт, пока не проиграл; сервер ждёт, пока не придут данные.

**Зачем `while`:** «крути, **пока** условие истинно» — без заранее известного `range`. Так делают проверку ввода, игровые циклы, ожидание ответа от API (позже).

**Отличие от `for`:** `for` — «ровно столько-то шагов»; `while` — «пока не случится нужное».

---

## Два вида циклов

| | `for` | `while` |
|---|--------|---------|
| Когда | **сколько раз** уже известно | повторять, **пока** условие истинно |
| Пример | числа 1…10, `range` | «ещё раз, пока не получится» |
| Ты уже знаешь | день 8–9 | сегодня |

`for` перебирает готовую последовательность.  
`while` перед **каждым** повтором проверяет условие: истинно — ещё круг, ложно — выход.

---

## Синтаксис `while`

```python
while условие:
    тело цикла
```

- **`while`** — «пока»
- **условие** — как у `if`: сравнение, результат `True` или `False`
- **`:`** и **отступ** — как у `for` и `if`

---

## Как работает цикл (на абстрактном примере)

```python
x = 1
while x <= 8:
    print(x)
    x = x * 2
```

Вывод: `1`, `2`, `4`, `8`.

Пошагово:

1. `x = 1`, условие `1 <= 8` — да → печатаем, `x` становится 2
2. `2 <= 8` — да → печатаем, `x` = 4
3. `4 <= 8` — да → печатаем, `x` = 8
4. `8 <= 8` — да → печатаем, `x` = 16
5. `16 <= 8` — **нет** → цикл кончился

Обрати внимание: в теле цикла **меняется** `x`. Без этого цикл не закончится.

---

## Главное правило: менять то, от чего зависит условие

```python
# ОПАСНО — x не меняется → бесконечный цикл
x = 1
while x <= 8:
    print(x)
```

Внутри `while` почти всегда есть строка, которая **двигает** задачу к выходу: прибавление, умножение, новый `input`, счётчик…

Если программа «зависла» — **Ctrl+C** в терминале.

---

## `for` vs `while` — когда что

**`for`** — число шагов известно заранее (`range`, список…).

**`while`** — «пока не…»:

- пока пользователь не ввёл правильный ответ;
- пока значение не вышло за предел;
- пока не нажали «стоп» (0, пустая строка и т.д.).

Пример с **повторным вводом** (другая ситуация, не из задач урока):

```python
answer = input("Кодовое слово: ")
while answer != "секрет":
    print("Неверно, попробуй ещё")
    answer = input("Кодовое слово: ")
print("Верно!")
```

Здесь условие про **строку** и `!=`. В задачах урока будут **другие** условия — подумай, что проверять и что менять внутри цикла.

---

## Что можно делать внутри `while`

Всё из прошлых дней:

- `input`, `int()`, `print`
- `if` / `elif` / `else`
- арифметика, сравнения, `and`
- переменная-счётчик или накопитель **до** цикла, изменение **внутри**

Теория **не подсказывает**, как именно собрать это в задачах — только инструменты. Связать их — твоя работа.

---

## Частые ошибки

| Ошибка | Что происходит |
|--------|----------------|
| не менять переменную в условии | бесконечный цикл |
| `while n > 0:` без начального `n` | `NameError` |
| забыть отступ у тела | `IndentationError` или код вне цикла |
| путать `=` и `==` в условии | не то поведение |
| проверка условия только один раз до цикла | цикл не повторяет ввод, когда нужно |
