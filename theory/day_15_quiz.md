# День 15 — мини-викторина (сборка пройденного)

## Зачем такие дни

**Проблема:** темы по отдельности («сегодня only `while`») редко встречаются в жизни по одной. Реальная программа — **склейка**: ввод, проверка, цикл, функция, строки.

**Зачем мини-проект каждые ~5 дней:** одна **цельная** программа из **уже пройденного**, без новых библиотек. Как пробный экзамен без новой воды — только то, что уже делал в A/B и D.

**Викторина** — классический формат: спросить → сравнить ответ → посчитать результат. Так устроены тесты, опросы, боты с кнопками «да/нет», проверка пароля.

---

## Из чего собрать викторину (только пройденное)

| Нужно | Откуда |
|-------|--------|
| текст вопроса, ответ пользователя | `input`, строки |
| «верно / неверно» | `if`, `==`, `.strip()`, `.lower()` |
| несколько вопросов подряд | `for` или повтор блоков |
| счёт правильных | переменная + `count = count + 1` |
| куски в функции | `def`, `return` |
| «пока не…» | `while` (опционально) |
| несколько условий | `and`, `or` |

**Списки и файлы пока не нужны** — вопросы можно задать тремя (или больше) блоками подряд или циклом с `if` по номеру шага.

---

## Два подхода (выбери сам)

**1. Три блока подряд** — проще думать, чуть больше повторов в коде:

```
вопрос 1 → проверка → +1 к счёту если верно
вопрос 2 → ...
вопрос 3 → ...
итог
```

**2. Функция «один вопрос»** — меньше копипасты:

```
def check_answer(правильный, ответ пользователя):
    return ...   # True/False или 1/0 — сам решишь

# три раза вызвать с разными вопросами
```

Оба подхода нормальны для дня 15.

## Сравнение ответов

Пользователь вводит `"  Да  "`, эталон `"да"`:

```python
user.strip().lower() == "да"
```

Числовой ответ — через `int()` до сравнения.

---

## Что проверить перед «готово»

- Программа **запускается** целиком, не по кускам.
- Минимум **3 вопроса**.
- В конце — **счёт** (сколько верных из скольки).
- Хотя бы **одна функция** (`def`).
- Нет тем, которых **ещё не было** (файлы, списки, import сторонних библиотек).
