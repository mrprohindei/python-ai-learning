# День 13 — функции (`def`)

## Зачем это придумали

**Проблема:** программы растут до тысяч строк. Один и тот же код («проверить пароль», «посчитать сумму», «спросить число») копируют в десяти местах. Ошибку исправляют в одном месте — в девяти забывают. Код становится нечитаемым: непонятно, *что* делает кусок, только *как*.

**Зачем `def`:** дать **имя** смысловому действию и написать его **один раз**. Вызов `ticket_price(age)` сразу говорит «считаем цену билета» — не нужно каждый раз читать десять строк `if`.

**В больших проектах** (сайт, бот, игра) почти всё живёт в функциях: «отправить сообщение», «прочитать файл», «спросить пользователя». Без функций к ИИ-приложениям не подобраться — там десятки таких кирпичиков.

---

## Зачем функции (в твоём коде сейчас)

Один и тот же код часто повторяется:

```python
print("---")
print("Меню")
print("---")
# ... позже снова те же три строки
```

**Функция** — именованный блок кода: написал **один раз**, вызываешь **сколько угодно**.

Позже, когда программы станут длиннее, функции — главный способ не утонуть в повторах.

---

## Синтаксис

```python
def имя():
    тело функции
```

```python
def say_hi():
    print("Привет!")
```

**Вызов** — по имени со скобками:

```python
say_hi()
say_hi()   # второй раз — без копирования тела
```

- `**def**` — «определяем функцию»
- **имя** — как у переменной: латиница, цифры, `_`
- `**()`** — список параметров (пока может быть пустым)
- `**:**` и **отступ** — как у `if` и `for`

---

## Параметры

Данные **внутрь** функции передают через параметры:

```python
def show_pet(animal):
    print("У меня есть", animal)

show_pet("кот")
show_pet("попугай")
```

`animal` — **параметр**. При вызове `show_pet("кот")` внутри функции `animal = "кот"`.

Несколько параметров — через запятую:

```python
def show_pair(a, b):
    print(a, "и", b)

show_pair(10, 20)
```

## `return` — отдать результат

`print` только **показывает** в терминале.  
`**return`** **возвращает** значение тому, кто вызвал функцию:

```python
def double(x):
    return x * 2

result = double(5)
print(result)   # 10
```

Без `return` функция отдаёт «ничего» (`None`) — для расчётов это обычно не то, что нужно.

```python
def add(a, b):
    return a + b

total = add(3, 4)
print(total)   # 7
```

## Где писать функции

Обычно **сначала** `def` (определения), **потом** основной код, который их **вызывает**:

```python
def greet(name):
    print("Здравствуй,", name)

greet("Сергей")
name = input("Имя: ")
greet(name)
```

Python читает файл сверху вниз, но **вызов** срабатывает только когда до него дошла очередь.  
Функцию можно определить **выше** первого вызова.

---

## Функция + то, что уже знаешь

Внутри функции работают `if`, `while`, `for`, `input`, `return` — как в обычном блоке кода.

```python
def is_warm(temp):
    if temp > 20:
        return True
    return False
```

`True` и `False` — логические значения («да» / «нет»). Их можно возвращать и проверять в `if`.

## Частые ошибки


| Ошибка                                  | Что происходит                      |
| --------------------------------------- | ----------------------------------- |
| забыли `:` после `def`                  | `SyntaxError`                       |
| вызов **до** `def` без определения выше | `NameError`                         |
| путают `print` и `return`               | функция «молчит», переменная пустая |
| нет отступа у тела                      | `IndentationError`                  |
| `def add(a, b)` но вызов `add(5)`       | не хватает аргумента                |
