# День 10 — мини-калькулятор с условиями

## Зачем это придумали (контекст урока)

**Проблема:** отдельные `if` для каждой операции (`+`, `-`, `*`, `/`) легко превращаются в путаницу — срабатывает не та ветка или несколько сразу. Деление на ноль **ломает** программу, если не перехватить.

**Зачем `elif` и проверки:** выбрать **ровно одно** действие из нескольких и **заранее** отсечь невозможное (деление на 0, неверная команда). Так устроены меню, кнопки, команды в ботах.

Этот день — **сборка** уже пройденного в одну программу, как в реальном маленьком проекте.

---

## Что за урок

Каждый **5-й** урок курса — **мини-проект**: одна программа из тем, которые уже прошли.  
Без новых библиотек. Код пишешь **ты**.

Сегодня — калькулятор: два числа, выбор операции, результат.  
Плюс **условия** — когда делать нельзя или операция неизвестна.

---

## Что уже пригодится


| Тема               | Зачем                                           |
| ------------------ | ----------------------------------------------- |
| `input`, `int()`   | спросить числа                                  |
| `+`, `-`, `*`, `/` | посчитать                                       |
| `if` / `else`      | проверки (деление на ноль, неверная операция)   |
| `==`               | сравнить введённый символ с `"+"`, `"-"` и т.д. |
| `print`            | вывести результат                               |


Цикл `for` в базовой версии **не обязателен** — одна операция за запуск.

---

## Новое: цепочка `if` / `elif` / `else`

В день 4 ты писал несколько отдельных `if`. Для **выбора одного из вариантов** удобнее цепочка:

```python
op = input("Операция (+, -, *, /): ")

if op == "+":
    print("Сложение")
elif op == "-":
    print("Вычитание")
elif op == "*":
    print("Умножение")
elif op == "/":
    print("Деление")
else:
    print("Неизвестная операция")
```

- `**elif**` — сокращение от «else if»: «иначе, если…»
- Срабатывает **только первая** подходящая ветка
- `**else`** в конце — ни одно условие не подошло

Сравнение со строкой:

```python
if op == "+":    # op — строка из input, "+" тоже в кавычках
```

---

## Схема калькулятора

Порядок действий:

1. Спросить **первое число** → `int()`
2. Спросить **второе число** → `int()`
3. Спросить **операцию** → строка (`"+"`, `"-"`, …)
4. **Проверить** особые случаи (деление на ноль)
5. **Посчитать** и вывести результат

---

## Условие: деление на ноль

Делить на `0` нельзя. Проверка **до** деления:

```python
if op == "/" and b == 0:
    print("Ошибка: на ноль делить нельзя!")
```

Слово `**and**` — оба условия должны быть истинны: выбрано деление **и** второе число равно нулю.

Можно и вложенно:

```python
if op == "/":
    if b == 0:
        print("Ошибка: на ноль делить нельзя!")
    else:
        print("Результат:", a / b)
```

Выбери тот вариант, который понятнее — оба рабочие.

---

## Пример логики (не готовая программа целиком)

```python
a = int(input("Первое число: "))
b = int(input("Второе число: "))
op = input("Операция (+, -, *, /): ")

if op == "+":
    print("Результат:", a + b)
elif op == "-":
    print("Результат:", a - b)
# ... остальные операции и else для неизвестной
```

Результат лучше выводить **понятно**: что за числа, какая операция, чему равен ответ.

---

## Частые ошибки


| Ошибка                                | Что происходит                                                 |
| ------------------------------------- | -------------------------------------------------------------- |
| `if op = "+"`                         | нужно `==`, не `=`                                             |
| забыли кавычки у `"+"`                | сравниваешь с несуществующей переменной                        |
| деление без проверки `b == 0`         | ошибка или `ZeroDivisionError`                                 |
| несколько отдельных `if` для операций | может сработать больше одной ветки (для операций лучше `elif`) |


---

## Критерий «готово»

Программа:

- спрашивает два числа и операцию;
- считает `+`, `-`, `*`, `/`;
- при делении на ноль — **сообщение об ошибке**, а не падение;
- при неизвестной операции — вежливое сообщение (`else`);
- запускается из терминала без сюрпризов.
