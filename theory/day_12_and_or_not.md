# День 12 — `and`, `or`, `not`

## Зачем это придумали

**Проблема:** правила в жизни и в программах **составные**: «пускать, если взрослый **и** с билетом»; «ошибка, если слишком холодно **или** слишком жарко». Одного `if x > 5` мало — получилась бы каша из вложенных `if`.

**Зачем `and` / `or` / `not`:** собирать **логику** из кусочков, как фразу из слов. Так описывают права доступа, фильтры, валидацию форм, условия в играх.

**В коде** это основа почти каждой проверки «можно / нельзя».

---

## Зачем (в твоих задачах)

Иногда одного сравнения мало:

- пускать, если **и** возраст подходит, **и** есть билет;
- предупреждать, если **холодно** **или** **жарко**;
- повторять ввод, пока **не** введут правильно.

Для этого в Python есть **логические операторы**. `and` ты уже применял в калькуляторе (день 10), `or` — в проверке возраста (день 11). Сегодня — системно, плюс **`not`**.

---

## `and` — оба условия истинны

```python
age = 20
has_ticket = True

if age >= 18 and has_ticket:
    print("Можно войти")
```

| age | has_ticket | age >= 18 and has_ticket |
|-----|------------|--------------------------|
| 20 | True | True → войти |
| 15 | True | False |
| 20 | False | False |

**`and`** — «и то, и другое». Одно `False` — всё выражение **ложно**.

---

## `or` — хотя бы одно истинно

```python
day = "суббота"

if day == "суббота" or day == "воскресенье":
    print("Выходной")
```

| day | результат |
|-----|-----------|
| суббота | True |
| понедельник | False |

**`or`** — «или первое, или второе, или оба». Оба `False` — всё выражение **ложно**.

---

## `not` — отрицание

```python
is_raining = False

if not is_raining:
    print("Зонт не нужен")
```

**`not`** переворачивает смысл: `not True` → `False`, `not False` → `True`.

С числами чаще пишут сравнение напрямую, но `not` удобен для флагов и «пока **не**…».

---

## В `if` и в `while`

Логические операторы работают **внутри условия**:

```python
temp = 45

if temp < -30 or temp > 40:
    print("Экстремальная погода!")
```

```python
answer = input("Согласен? (да/нет): ")
while answer != "да" and answer != "нет":
    print("Введи да или нет")
    answer = input("Согласен? (да/нет): ")
```

Условие читается как обычная фраза: «если холодно **или** жарко», «пока **не** да **и не** нет».

---

## Скобки для ясности

Когда условие длинное, скобки помогают:

```python
if (age >= 6 and age <= 11) or age >= 65:
    print("Льготный тариф")
```

Без скобок Python тоже понимает, но скобки делают код читаемее для человека.

---

## Частые ошибки

| Ошибка | Проблема |
|--------|----------|
| `if a and b > 5` — имелось «оба > 5» | нужно `if a > 5 and b > 5` |
| путать `and` и `or` | «или» пропускает чаще, «и» — строже |
| `if not x > 5` | лучше `if x <= 5` — понятнее |
| `=` вместо `==` в условии | присваивание вместо сравнения |

## Как это связано с прошлым

| Уже знаешь | Сегодня |
|------------|---------|
| `if` / `elif` / `else` | несколько условий в одном `if` |
| `while` | условие с `and` / `or` |
| `==`, `<`, `>` | части большого условия |
