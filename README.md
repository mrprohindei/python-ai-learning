# Python AI Learning

Личный проект для обучения Python с прицелом на локальные ИИ-приложения (чат, RAG, агенты) в течение ~года.

---

## Как устроено обучение

### Принципы (обязательно соблюдать в каждом чате)

1. **Без спешки** — одна тема может занимать несколько дней.
2. **Ученик пишет код сам** — ИИ не даёт готовое решение сразу; сначала попытка ученика, потом разбор.
3. **Следующий урок только после проверки** — переход дальше, когда 2–3 задачи по теме сделаны уверенно и проверены в чате.
4. **Мало теории (5–10 мин), много практики** — теория в чате, не в файлах.
5. **Тема не усвоена → не новая тема**, а ещё задачи или разбор ошибок.
6. **Спокойный темп** — не торопить, не перескакивать через темы.
7. **Cursor — помощник, не замена** — цель: понимать код, а не только генерировать.

Подробные правила для ИИ-наставника: **`learning_rules.md`**

### Формат одного дня (~60–90 мин)

1. Короткое вступление в чате (1 идея + 1 пример) ~10 мин
2. Задача A → ученик пишет → проверка в чате
3. Задача B → ученик пишет → проверка в чате
4. Опционально задача C или домашка

**Критерий перехода:** 2–3 задачи сделаны учеником, проверены, явное «тема закрыта».

### Правило «объясни своими словами»

Перед закрытием темы ученик отвечает в чате:

- что делает код;
- какие новые команды использовались;
- где ошибался и как исправил;
- как сам проверил результат.

Без этого — тему не закрываем. Подробнее в `learning_rules.md`.

### Формат файлов уроков

- В `lessons/day_XX.py` — **только формулировка задач** в общих чертах, без подсказок по синтаксису.
- Подсказки, теория, разбор ошибок — **в чате**, по запросу ученика.
- Меньше подсказок в файлах → больше самостоятельного мышления.

### Как работать в Cursor

1. Открыть папку `python-ai-learning` как workspace (**File → Open Folder**).
2. Редактировать `lessons/day_XX.py`.
3. Сохранить (**Ctrl+S**).
4. Терминал (**Ctrl+`**) → запуск:

   ```powershell
   cd "путь/к/python-ai-learning"
   python lessons/day_XX.py
   ```

5. Проверка в чате — по шаблону из **`lesson_check_template.md`**.

### Что писать в новом чате для продолжения

```
Открой python-ai-learning. Прочитай README.md, progress.md, learning_rules.md.
Продолжи с текущего дня. Соблюдай правила наставника.
```

## Как сделать обучение эффективнее

### Сначала пробуй сам

- Прочитай задачу в `lessons/day_XX.py`.
- Напиши код без автодополнения «под ключ» от ИИ.
- Запусти в терминале, посмотри результат или текст ошибки.
- 10–15 минут самостоятельной попытки — норма, не провал.

### Когда просить подсказку

- После своей попытки, если **застрял больше ~15 минут** на одном месте.
- Если **ошибка в терминале** непонятна — скопируй её в чат и напиши, что пробовал.
- Если код **работает**, но результат **странный** — опиши, что ожидал и что получил.

Формулировка: «Я пробовал X, получил Y, не понимаю Z» — не «напиши решение».

### Как правильно просить проверку

Используй **`lesson_check_template.md`**: номер дня, `@lessons/day_XX.py`, список пунктов для проверки.

### Почему нельзя сразу просить готовое решение

- Мозг запоминает то, что **сам написал и отладил**.
- Готовый код из чата создаёт иллюзию «я умею», пока на следующей задаче всё снова ломается.
- Цель года — **писать приложения**, а не копировать фрагменты.

ИИ может дать решение **после твоей попытки** — только для разбора, не для вставки в урок.

### Как понять, что тема усвоена

- 2–3 задачи сделаны **без подсказок по синтаксису**.
- Код **запускается**, ты **объясняешь его своими словами**.
- Типичные ошибки темы ты **узнаёшь до запуска** (см. `mistakes.md`).
- Контрольные вопросы наставника — **уверенные ответы**.

### Cursor / Composer как наставник, а не автогенератор

- **Хорошо:** «Почему тут TypeError?», «Проверь день 3», «Объясни, что делает `int()`».
- **Плохо:** «Напиши весь day_03.py», «Сделай калькулятор за меня».
- Автодополнение Tab — для **одной строки**, которую ты уже понимаешь, не для целого урока.
- Ошибки заноси в **`mistakes.md`** — это твой личный справочник.
