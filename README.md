# Python AI Learning

Личный проект для обучения Python с прицелом на локальные ИИ-приложения (чат, RAG, агенты) в течение ~года.

**«День» в файлах** — одна тема (~75–100 мин), не календарный день. За вечер можно пройти 2–4 темы — это нормально.

---

## Как устроено обучение

### Принципы (обязательно соблюдать в каждом чате)

1. **Без спешки** — одна тема может занимать несколько дней.
2. **Ученик пишет код сам** — ИИ не даёт готовое решение сразу; сначала попытка ученика, потом разбор.
3. **Следующий урок только после проверки** — переход дальше, когда задачи **A и B** сделаны уверенно и проверены в чате.
4. **Теория + практика** — файл в `theory/` (~20–25 мин) + задачи; **теория ≠ готовые решения задач**.
5. **Тема не усвоена → не новая тема**, а ещё задачи или разбор ошибок.
6. **Спокойный темп** — не торопить, не перескакивать через темы.
7. **Cursor — помощник, не замена** — цель: понимать код, а не только генерировать.

Подробные правила для ИИ-наставника: **`learning_rules.md`**

### Формат одного дня (~75–100 мин)

1. **Теория** (~20–25 мин) — `theory/day_XX_*.md` (есть блок **«зачем придумали»**)
2. **Резюме** в чате — главная идея + вопросы
3. **Задача A** → ученик пишет → проверка
4. **Задача B** → ученик пишет → проверка
5. Опционально **задача C** (чуть сложнее по теме дня)
6. Опционально **задача D** — в **`problem_D/day_XXD.py`** (сборка из прошлых тем; можно в любое время)

В задачах есть строки `# Проверь: вход → выход` — прогоняй их сам **до** проверки в чате.

| Задача | Где | Для закрытия дня |
|--------|-----|------------------|
| A, B | `lessons/day_XX.py` | **обязательны** |
| C | `lessons/day_XX.py` | по желанию |
| D | `problem_D/day_XXD.py` | по желанию, не блокирует |

**Критерий перехода:** A и B проверены + «объясни своими словами» в чате.

### Правило «объясни своими словами»

Перед закрытием темы — в чате: что делает код; новые команды; где ошибался; как проверил. Подробнее в `learning_rules.md`.

### Формат файлов

| Папка / файл | Содержание |
|--------------|------------|
| `theory/` | теория к каждому дню (читать **до** задач) |
| `lessons/day_XX.py` | задачи A, B, C |
| `problem_D/day_XXD.py` | задача D — сборка, отдельный файл на урок |
| `шпаргалка.md` | краткий справочник по пройденному |
| `progress.md` | **живой** статус: текущий день, история |
| `mistakes.md` | личный журнал ошибок |
| чат | резюме, подсказки по запросу, проверка |

Подсказки к задачам — **по запросу** в чате, не в файлах уроков.

### Как работать в Cursor

1. **File → Open Folder** → папка `python-ai-learning`.
2. Редактировать `lessons/day_XX.py` (и при желании `problem_D/day_XXD.py`).
3. **Ctrl+S** — сохранить.
4. Терминал (**Ctrl+`**):

   ```powershell
   cd "C:\python-ai-learning"
   python lessons/day_XX.py
   ```

5. Проверка — шаблон в **`lesson_check_template.md`**.

### Что писать в новом чате

```
Открой python-ai-learning. Прочитай README.md, progress.md, learning_rules.md.
Продолжи с текущего дня. Соблюдай правила наставника.
```

## Как сделать обучение эффективнее

- **10–15 минут** самостоятельной попытки перед подсказкой — норма.
- Формулировка: «Я пробовал X, получил Y, не понимаю Z» — не «напиши решение».
- При проверке: «Проверь день X» + `@lessons/day_XX.py` (и `@problem_D/day_XXD.py` для D).
- Ошибки — в **`mistakes.md`**. Синтаксис — в **`шпаргалка.md`**.

## Git и GitHub ✅

| | |
|---|---|
| **Remote** | `https://github.com/mrprohindei/python-ai-learning.git` |
| **Ветка** | `main` |

Git **не копирует папку целиком** — только то, что ты **add → commit → push**. Команды выполняй **из папки проекта**:

```powershell
cd "C:\python-ai-learning"
git status
git add .
git commit -m "День 14: строки, 14D"
git push
```

**На другом ПК (перед уроком):**

```powershell
cd "C:\python-ai-learning"
git pull
```

**После урока:** `git add .` → `commit` → `push`.

**В Cursor:** Source Control (**Ctrl+Shift+G**) — то же самое без терминала.

**Не попадает на GitHub** (и не должно): `__pycache__/`, `venv/`, `.env` — см. `.gitignore`.
